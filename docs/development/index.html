Running Webservice in Docker (Development)â€‹

caution

Support running this template webservice in Docker is NOT for production deployment. It's intended usage is for development where developer can easily stand up a webservice instance for dev and testing purpose.

Production deployment for astraios assumes Jetty-based scheme

Getting Imageâ€‹

Docker Hubâ€‹

We can pull the image from its docker hub:

docker pull jack20191124/astraios

GitHubâ€‹

We could also build the image from source:

https://github.com/paion-data/astraios.git
cd astraios
docker build -t paiondata/astraios
tip

If we need to pass one or more runtime environment variables into docker image, we can do

export MY_ENV_VARIABLE=foo
export MY_OTHER_ENV_VARIABLE=bar

docker build -t paiondata/astraios \
  --build-arg MY_ENV_VARIABLE=$MY_ENV_VARIABLE \
  --build-arg MY_OTHER_ENV_VARIABLE=$MY_OTHER_ENV_VARIABLE \
  .

where the Dockerfile contains

ARG MY_ENV_VARIABLE
ARG MY_OTHER_ENV_VARIABLE

ENV MY_ENV_VARIABLE $MY_ENV_VARIABLE
ENV MY_OTHER_ENV_VARIABLE $MY_OTHER_ENV_VARIABLE

Standup a Containerâ€‹

When image is on our machine (either by pulling or building), we can spin up an instance using

docker run -d --name=astraios -p 8080:8080 paiondata/astraios

At this moment, the healthcheck endpoint GET localhost:8080/v1/data/healthcheck should return 200 status code properly.

Running Webservice in Standalone Jetty (Production)â€‹

Download Jettyâ€‹

At download page, pick up a .tgz distribution. It is very important to pick up Jetty server version that matches JDK version. For JDK 17, it's been tested that Jetty 11.0.15 works

note

During testing, the embedded Jetty version is also 11.0.15

Hence, we will use "11.0.15" release as an example:

Put the tar.gz file into a location of your choice as the installation path and extract the Jetty binary using

tar -xzvf jetty-home-11.0.15.tar.gz

The extracted directory jetty-home-11.0.15 is the Jetty distribution. We call this directory $JETTY_HOME, which should not be modified.

Setting Up Standalone Jettyâ€‹

Our WAR file will be dropped to a directory where Jetty can pick up and run. We call this directory $JETTY_BASE, which is usually different from the $JETTY_HOME. The $JETTY_BASE also contains container runtime configs. In short, the Standalone Jetty container will be setup with

export JETTY_HOME=/path/to/jetty-home-11.0.15
mkdir -p /path/to/jetty-base
cd /path/to/jetty-base
java -jar $JETTY_HOME/start.jar --add-module=annotations,server,http,deploy

where /path/to/ is the absolute path to the directory containing the jetty-home-11.0.15 directory

The --add-module=annotations,server,http,deploy is how we configure the Jetty container.

Lastly, drop the WAR file into /path/to/jetty-base/webapps directory and rename the WAR file to ROOT.war:

mv /path/to/war-file /path/to/jetty-base/webapps/ROOT.war

Running Astraiosâ€‹

java -jar $JETTY_HOME/start.jar

The webservice will run on port 8080
